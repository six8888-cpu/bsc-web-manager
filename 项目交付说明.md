## 📖 使用流程

### 第一次使用 (5分钟)

1. **启动Web服务**
   - Windows: 双击 `start_windows.bat`
   - Linux: 运行 `./deploy/start.sh`

2. **访问Web界面**
   - 打开浏览器: `http://localhost:5000`

3. **连接B端服务器**
   ```

   端口: 22
   用户名: root

   ```

4. **配置生成参数**
   ```
   前缀: 88
   后缀: (留空)
   包含: (留空)
   生成数量: 1
   CPU核心: 拖动滑块选择
   ```

5. **开始生成**
   - 点击"开始生成"
   - 观察实时输出
   - 等待完成

6. **下载结果**
   - 点击"下载钱包文件"
   - 保存到安全位置

### 3. 灵活的匹配规则

**单独使用：**
- 前缀: `1780` → `0x1780...`
- 后缀: `8888` → `0x...8888`
- 包含: `666` → `0x...666...`

**组合使用：**
- 前缀 + 后缀: `0x1780...8888`
- 前缀 + 包含: `0x1780...666...`
- 全部组合: `0x1780...666...8888`

**大小写：**
- 不区分: `abc` = `ABC` = `AbC`
- 区分: `abc` ≠ `ABC`

### 4. 智能CPU调度

**自动检测：**
- 连接成功后自动获取CPU核心数
- 默认使用 `核心数-1` (保留1核给系统)

**拖拽选择：**
- 滑块拖动选择使用的核心数
- 实时显示当前值
- 范围: 1 ~ 最大核心数

**性能对比：**
```

8核  →  8-12万地址/秒
16核 →  18-25万地址/秒
```

## 🐛 常见问题

### Q1: 连接失败怎么办？

**检查项：**
1. 服务器IP是否正确
2. SSH端口是否开放 (默认22)
3. 用户名密码是否正确
4. 防火墙是否允许SSH

**解决方案：**
```bash
# 在B端服务器执行
systemctl status sshd
ufw allow 22/tcp
```

### Q2: 依赖安装失败？

**原因：** 网络问题或pip源慢

**解决：** 系统已自动使用阿里云镜像源，一般会成功

### Q3: 生成速度慢？

**检查：**
- CPU核心数是否调到最大？
- 服务器配置是否太低？
- 难度是否太高？

**优化：**
- 拖动滑块到最大值
- 升级服务器配置
- 降低难度 (减少位数)

### Q4: 可以同时运行多个任务吗？

**可以！** 但建议：
- 一台服务器一次运行一个任务
- 避免资源竞争影响速度

### Q5: 支持哪些系统？

**Web端 (香港服务器):**
- ✅ Ubuntu 20.04+
- ✅ Debian 11+
- ✅ CentOS 7/8
- ✅ Windows 10+ (本地测试)

**B端 (运行生成器):**
- ✅ Ubuntu 20.04+
- ✅ Debian 11+
- ✅ CentOS 7/8
- ❌ Windows (不支持)
